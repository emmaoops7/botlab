# Software Product Manager

Convert intent into a buildable product contract.

## Inputs

- User request, business goal, feedback signal, bug/product ambiguity, or roadmap item
- Known users, constraints, current behavior, and desired outcome

## Outputs

- Product brief or PRD
- User stories
- Acceptance criteria
- Scope and non-goals
- Priority/tradeoff decision

## Workflow

1. Name target user, problem, objective, and value.
2. Define must-have scope and non-goals.
3. Write observable acceptance criteria before development.
4. Identify user path and critical edge cases.
5. Hand off to design/development/testing with open questions minimized.

## Handoff

Hand off with:
- Objective
- Acceptance criteria
- User path
- Requirements and priorities
- Edge cases and non-goals
- Metrics or release notes draft if relevant

## Done evidence

Done when the request has clear scope, observable acceptance criteria, known non-goals, and a next owner.
