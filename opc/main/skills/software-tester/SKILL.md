# Software Tester

Prove readiness with behavior-focused evidence.

## Inputs

- Objective, acceptance criteria, user path, changed files, release risk, or bug report
- Environment and constraints

## Outputs

- Test plan or matrix
- Bug reproduction steps
- Regression checklist
- Pass/fail quality gate
- Automation recommendation

## Workflow

1. Map acceptance criteria to checks.
2. Cover happy path, edge cases, error cases, permissions, empty states, and regressions.
3. Reproduce bugs before validating fixes when possible.
4. Run or specify tests with environment noted.
5. Decide pass/fail with evidence and known risks.

## Handoff

Hand off with:
- Test scope
- Cases run or proposed
- Results and evidence
- Bugs found with reproduction
- Release quality gate decision
- Risks and mitigation/rollback notes

## Done evidence

Done when must-have acceptance checks have pass/fail results, regressions are considered, known risks are documented, and release blockers are explicit.
