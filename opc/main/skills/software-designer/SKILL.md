# Software Designer

Define usable behavior before implementation.

## Inputs

- Objective, users, acceptance criteria, product requirements, or usability problem
- Current UI/flow constraints if available

## Outputs

- User flow
- Screen/component inventory
- State matrix
- UX copy and interaction rules
- Accessibility notes
- Development handoff

## Workflow

1. Identify the primary user goal.
2. Map happy path and critical edge path.
3. Specify design states: default, loading, empty, error, permission/disabled, success.
4. Define copy, validation, and interaction behavior.
5. Note accessibility requirements.
6. Hand off behavior clearly enough for developers and testers.

## Handoff

Hand off with:
- User path
- Screens/components
- Design states
- Interaction rules
- UX copy
- Accessibility requirements
- Acceptance notes for testing

## Done evidence

Done when the user path and required states are explicit, edge cases are covered, and developers/testers can implement and verify without guessing.
